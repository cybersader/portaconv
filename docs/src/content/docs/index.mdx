<Aside type="note" title="What paths are NOT rewritten">
`Conversation.cwd` and per-message `extensions` are metadata about the authoring environment — they describe the session, not its content. They're preserved verbatim so you can still see where the session ran. Only text blocks + tool-call args + tool-result bodies are rewritten.
</Aside>

## Commands (v0.1)

<CardGrid>
  <Card title="pconv list" icon="list-format">
    List conversations from any detected agent CLI. Filter by
    workspace, time, or tool. Table or JSON output.
  </Card>
  <Card title="pconv dump" icon="document">
    Dump one session to stdout. Default output is paste-ready
    markdown with tool calls and truncated results. Opt-in flags:
    `--rewrite`, `--include-thinking`, `--full-results`.
  </Card>
  <Card title="pconv mcp serve" icon="seti:config">
    Stdio MCP server (coming soon). `list_conversations` +
    `get_conversation` tools; each conversation exposed as
    `convos://conversation/&lt;id&gt;`.
  </Card>
</CardGrid>

## Design principles

<div style="display: grid; grid-template-columns: repeat(auto-fit, minmax(180px, 1fr)); gap: 0.75rem; margin: 1.5rem 0;">
  <div style="border: 1px solid var(--sl-color-gray-4); border-radius: 10px; padding: 0.75rem 1rem; background: var(--sl-color-gray-6);">
    <div style="font-weight: 700; margin-bottom: 0.3rem;">Read-only</div>
    <div style="font-size: 0.8rem; opacity: 0.8;">Never writes to any tool's storage. On-demand reads. No daemon, no file watching.</div>
  </div>
  <div style="border: 1px solid var(--sl-color-gray-4); border-radius: 10px; padding: 0.75rem 1rem; background: var(--sl-color-gray-6);">
    <div style="font-weight: 700; margin-bottom: 0.3rem;">Resilient</div>
    <div style="font-size: 0.8rem; opacity: 0.8;">Unknown record types preserved in <code>extensions.unknown_records</code>. Adapter version drift doesn't crash the load.</div>
  </div>
  <div style="border: 1px solid var(--sl-color-gray-4); border-radius: 10px; padding: 0.75rem 1rem; background: var(--sl-color-gray-6);">
    <div style="font-weight: 700; margin-bottom: 0.3rem;">Composable</div>
    <div style="font-size: 0.8rem; opacity: 0.8;">Unix-pipe-friendly stdout. JSON for machine processing. Markdown for humans. No built-in clipboard — pipe to <code>clip.exe</code> / <code>pbcopy</code> yourself.</div>
  </div>
  <div style="border: 1px solid var(--sl-color-gray-4); border-radius: 10px; padding: 0.75rem 1rem; background: var(--sl-color-gray-6);">
    <div style="font-weight: 700; margin-bottom: 0.3rem;">Future-accounting</div>
    <div style="font-size: 0.8rem; opacity: 0.8;">OpenAI-shape core + extensions bag means new tools slot in without a schema migration.</div>
  </div>
</div>

## How it composes

portaconv is a sibling to [portagenty](https://github.com/cybersader/portagenty)
(the terminal-native workspace launcher). Use them together or
independently:

- **Standalone**: `cargo install portaconv`, run `pconv` from any
  terminal. Reads `~/.claude/projects/` directly.
- **With portagenty** _(planned)_: a `pa convos` shim passes workspace
  context to `pconv` so list/dump automatically scope to the
  current workspace's projects.
