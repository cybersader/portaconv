## Here's what you do

Four commands. That's the whole tool.

```sh
pconv list                          # see your recent sessions
pconv dump <session-id>             # render one as paste-ready markdown
pconv dump <session-id> | clip.exe  # WSL — copy to clipboard
# (use pbcopy on macOS, wl-copy on Wayland)

pconv doctor                        # detect stale sessions-index.json
pconv rebuild-index --all           # repair the /resume picker
```

Paste into a new Claude Code session, claude.ai, opencode, whatever
— the conversation keeps going. If the session was authored on a
different OS, add `--rewrite wsl-to-win` (or `win-to-wsl`) so paths
inside the content get translated on the way out. When the native
`/resume` picker shows wrong sessions, `pconv rebuild-index` fixes
the underlying index (upstream
[#25032](https://github.com/anthropics/claude-code/issues/25032)).

**Even faster, inside a portagenty workspace:**

```sh
# Skip the list + copy UUID dance — just grab the most recent
# session in this workspace and send it straight to the clipboard.
pconv dump --latest --workspace-toml auto | clip.exe

# Long session? Keep only the last 50 messages.
pconv dump --latest --workspace-toml auto --tail 50 | clip.exe
```

## Commands

<CardGrid>
  <Card title="pconv list" icon="list-format">
    List conversations discoverable on this machine. Filter by
    `--workspace-toml auto`, `--since 2d`, or `--grep refactor`;
    sort with `--sort msgs|title|started`; cap with `--limit`.
    Table or JSON output.
  </Card>
  <Card title="pconv dump" icon="document">
    Dump one session to stdout. `--latest --workspace-toml auto`
    skips the id-copy step. `--tail N` keeps only the last N
    messages. `--rewrite wsl-to-win|win-to-wsl|strip` translates
    paths. Default output: paste-ready markdown.
  </Card>
  <Card title="pconv doctor" icon="warning">
    Detect stale `sessions-index.json` under `~/.claude/projects/`.
    `--dump-stale` emits paste-ready markdown for the newest
    session in each stale project so you can recover into a fresh
    `claude` without waiting on the picker.
  </Card>
  <Card title="pconv rebuild-index" icon="approve-check">
    Rewrite `sessions-index.json` from the actual `.jsonl`s so the
    native `/resume` picker works correctly. Atomic write, dated
    `.bak-YYYY-MM-DD` backup. `--all --lag-threshold-hours 168`
    for weekly cron.
  </Card>
  <Card title="pconv mcp serve" icon="seti:config">
    Stdio MCP server. `list_conversations`, `get_conversation`,
    and `doctor` tools with typed schemas (every CLI flag mirrored),
    plus each conversation as a `convos://conversation/&lt;id&gt;`
    resource. [See wiring guide](/portaconv/concepts/agents-and-portagenty/).
  </Card>
</CardGrid>

<Aside type="note" title="What paths are NOT rewritten">
`Conversation.cwd` and per-message `extensions` are metadata about the authoring environment — they describe the session, not its content. Only text blocks + tool-call args + tool-result bodies are rewritten.
</Aside>

## What you can rely on

No daemon. No file watching. `pconv` never touches `.jsonl`
conversation content — the substrate of your Claude history is
never at risk. The only file `pconv` writes to is
`sessions-index.json` (via `rebuild-index`), atomically with a
dated `.bak-YYYY-MM-DD` backup by default. Close the process and
nothing's running. Every feature has explicit reliability tiers:
what's always the same, what's opt-in, what could drift as Claude
evolves. See the
[reliability tiers table](/portaconv/concepts/#reliability-tiers)
for the full breakdown — it's the first thing to read if you're
scripting against portaconv.

## How it composes

portaconv is a sibling to [portagenty](https://github.com/cybersader/portagenty)
(the terminal-native workspace launcher). Use them together or
independently:

- **Standalone**: `cargo install portaconv`, run `pconv` from any
  terminal. Reads `~/.claude/projects/` directly.
- **With portagenty** _(planned)_: a `pa convos` shim passes workspace
  context to `pconv` so list/dump automatically scope to the
  current workspace's projects.
